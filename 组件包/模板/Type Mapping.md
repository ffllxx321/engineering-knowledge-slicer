# Type Mapping

本文件只判断知识原子应使用“动态事件卡片”还是“静态信息卡片”。物理目录由 `library + folder_type` 决定，禁止使用 `Category / TagL1 / TagL2 / Map_Index` 推导落盘目录。

## 判断顺序

1. 内容是否描述了在某一时间发生、需要跟踪状态或责任人的事项？是则优先使用动态事件卡片。
2. 内容是否描述可跨项目复用的事实、规则、要求、参数、方法、对象或经验？是则使用静态信息卡片。
3. 同一来源可同时拆出动态与静态卡片。例如会议的“批准采用某方案”是动态决策，“该方案的技术参数”是静态知识。
4. 不得为了套模板生成空洞卡片。没有可验证的独立知识点时返回空数组。

## 动态事件类型

| Event_Type | 适用内容 | 必备信息 |
|---|---|---|
| `#type/meeting` | 会议、沟通、协调记录 | 背景、讨论事项、结论 |
| `#type/decision` | 已形成的决策或审批结论 | 决策内容、依据、影响 |
| `#type/issue` | 待解决问题、障碍、冲突 | 问题、影响、当前状态 |
| `#type/change` | 设计、范围、合同或现场变更 | 变更前后、原因、影响 |
| `#type/risk` | 新识别或状态变化的风险 | 风险事件、概率/影响、应对 |
| `#type/review` | 内审、评审、复核、检查 | 审查对象、发现、结论 |
| `#type/action` | 有责任人和期限的行动项 | 动作、责任人、期限 |
| `#type/approval` | 报批报建、许可、审批过程 | 申请事项、主管部门、状态 |
| `#type/claim` | 索赔、反索赔、争议处理 | 事由、依据、金额/工期影响 |
| `#type/inspection` | 验收、检测、巡检 | 对象、标准、结果 |
| `#type/nonconformance` | 不符合项、质量缺陷、整改 | 缺陷、原因、纠正措施 |
| `#type/schedule-delay` | 计划变化、延误、赶工 | 计划偏差、原因、措施 |
| `#type/procurement` | 采购、询价、定标、到货事件 | 标的、供应商、节点、结论 |
| `#type/client-request` | 客户需求、反馈、投诉 | 请求内容、影响、响应 |
| `#type/contract-negotiation` | 合同谈判与签约过程 | 条款分歧、共识、待办 |

## 静态信息类型

| Info_Type | 适用内容 | 典型原子 |
|---|---|---|
| `#info/spec` | 标准、规范、技术规格 | 适用范围和强制要求 |
| `#info/clause` | 招标、合同、法规条款 | 条款原文、解释、义务 |
| `#info/parameter` | 数值、阈值、容量、性能 | 参数、单位、条件 |
| `#info/requirement` | 业主、招标、设计、施工要求 | 要求、验收条件、约束 |
| `#info/entity` | 客户、项目、供应商、主管部门 | 属性、角色、关系 |
| `#info/process` | 固定流程、审批路径、作业程序 | 输入、步骤、输出 |
| `#info/method` | 技术方法、施工方法、分析方法 | 场景、步骤、限制 |
| `#info/strategy` | 营业、投标、谈判、采购策略 | 目标、做法、适用条件 |
| `#info/price` | 单价、费率、报价构成 | 口径、币种、有效条件 |
| `#info/case` | 项目案例、成功经验、失败教训 | 情境、做法、结果、启示 |
| `#info/capability` | 人员、团队、企业能力 | 能力项、证据、适用任务 |
| `#info/compliance` | 法规、许可、合规义务 | 主管部门、要求、时限 |
| `#info/schedule` | 基准计划、里程碑、逻辑关系 | 节点、前置条件、工期 |
| `#info/template` | 可复用模板、清单、话术 | 使用场景、字段、方法 |
| `#info/evidence` | 证书、业绩、检测报告等证据 | 证据对象、有效期、出处 |
| `#info/design-basis` | 设计依据、边界和假设 | 依据、假设、限制 |
| `#info/responsibility` | 组织职责、接口和分工 | 主体、职责、交付物 |

## 禁止规则

- 不把整份文件概述当作一个知识原子。
- 不生成只有“加强管理、持续优化、提高意识”等空泛表述的卡片。
- 不补写来源中不存在的责任人、日期、金额、参数或结论。
- 不因字段缺失而填入“待补充”“未知”“暂无”等正文占位语；缺失字段留空并由插件在渲染时省略。
