# Engineering Knowledge Slicer PRD v1.1 Rebuild Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild the installed Obsidian plugin so it follows PRD v1.1: API-only document parsing, MiniMax classification/summary/atomization, fixed business routing, stable identities, explainable confidence, resumable tasks, and batch exception review.

**Architecture:** Keep `main.js` as the Obsidian adapter and split business behavior into focused CommonJS modules under `src/core`. The component pack is the runtime source of truth: `folder-map.json`, prompts, templates, Tag Library, Type Mapping, and JSON Schemas. Every pipeline stage persists a versioned artifact and validates its input/output before advancing.

**Tech Stack:** Obsidian Desktop API, CommonJS JavaScript, Node test runner, MiniMax domestic OpenAI-compatible endpoint, MinerU precise parsing API, PaddleOCR jobs API, JSON Schema, deterministic Markdown templates.

## Target File Structure

New focused modules:

```text
src/core/
├── component-contracts.js   # folder map, prompt/template/schema loading and validation
├── pipeline.js              # stage orchestration and checkpoints
├── document-parser.js       # format routing and normalized parse packages
├── ai-pipeline.js           # MiniMax classification, summaries, atoms, repair
├── confidence.js            # P/T/E/S/A calculation and hard caps
├── identity.js              # source/run/card identity and atom fingerprints
├── markdown-renderer.js     # deterministic templates and escaping
├── link-service.js          # candidate search and semantic relation validation
└── migration.js             # settings/task ledger v3 migration
```

Existing modules remain only where they retain a clear responsibility; obsolete local parser and old MOC routing code are removed after replacement tests are green.
