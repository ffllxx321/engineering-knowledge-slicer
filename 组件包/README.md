# 工程知识切片组件包

## 内容

- `folder-map.json`：28 个固定目录及类型提示词的唯一机器可读路由。
- `schemas/`：任务、解析包、分类、结构化总结、知识原子和卡片 JSON Schema。
- `模板/Type Mapping.md`：判断静态信息或动态事件，不负责目录路由。
- `模板/静态信息卡片.md`：静态知识原子的 Markdown 渲染模板。
- `模板/动态事件卡片.md`：动态事件原子的 Markdown 渲染模板。
- `提示词/00-类型判定.md`：判定 `library + folder_type + document_type`。
- `提示词/01-结构化总结-基础.md`：所有类型共用的总结约束与 JSON 协议。
- `提示词/招投标/`、`提示词/业务库/`：固定目录对应的类型化总结提示词。
- `提示词/99-知识原子生成.md`：第二次 MiniMax 调用的知识原子协议。

## 调用顺序

1. MinerU/PaddleOCR 生成标准解析包。
2. 调用类型判定提示词。
3. 合并“结构化总结基础提示词 + 对应类型提示词”生成结构化总结。
4. 合并“知识原子生成提示词 + Type Mapping + Tag Library 白名单”生成原子 JSON。
5. 插件校验 JSON、检索关联候选，再用静态或动态模板渲染 Markdown。

模型不得直接自由拼接 frontmatter。模板中的可选区块仅在有可靠内容时渲染，禁止保留空占位字段。
